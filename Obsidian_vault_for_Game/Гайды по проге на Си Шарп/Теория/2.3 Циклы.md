Циклы нужны чтобы повторять одни и те же действия несколько раз.

Увеличивает или уменьшает переменную на 1 (Почитай про инкремент и декремент, если забыл что это. В чем разница префиксной и постфиксной формы) В нашем примере это не так важно, но помнить надо.
x++, 
++x,
x--, 
--x

```C#
int days = 0;

while(days < 365)
{
	Console.WriteLine("С начала года прошло" + days "дней");
	days++; // увеличивает переменную на 1
}

/*
Знакомься, это цикл While. Он п овторяет все что внутри него {}, до тех пор, пока условие () Верно(true)
Что происходит в нашем примере. Изначально у нас  0 дней.
1. Цикл проверяет 0 < 365 результат true, значит начинаем работу
2. Цикл выполняет все команды друг за другом, выводт надпись и САМОЕ ГЛАВНОЕ увеличивает дни на 1, было 0, станет 1
3. Все начинается с начала 
4. Проверка дней 1 < 365, да(true) Значит выполняем команды еще раз
5. Опять выводим надпись
6. опять увеличиваем дни на 1, было 1 станет 2 
7. И так еще 363 раза
   Когда days станет 365, произойдет последняя проверка 365 < 365 (False)
   Цикл не будет выполнять команды и программа завершиться
   
   
   ИТАК, цикл while(){} проверяет условие, если оно верно, выполнит одну итерацию команд. Вот и все. Главное не забывать увеличивать или уменьшать счетчик, а то цикл будет бесконечным
   
   
   Есть так же его собрат по разуму. do{} while(). Он сначала делает, то что сказали, а потом только сравнивает
*/
do
{
	Console.WriteLine("С начала года прошло" + days "дней");
	days++; 
} while (days < 365)

// ЦИКЛ For. Используется, когда мы точно знаем сколько раз надо выполнить цикл

for (int i = 0; i < 10; i++) 
{
	// for разделен на 3 блока, первый это создание счетчика (int i = 0), по сути просто создаем обычную числовую перменную, прямо в цикле. Второй блок - условия входа в цикл (прямо как у While. Пока i < 10 мы будем выполнять работу) Блок 3: изменение счетчика чтобы цикл не был бесконечным.
	// Буква i является стандартным обозначением для счетчика внутри for. Это наиболее частый вариант, но никто не мешает назвать эту перменную как душе угодно
}

// Перепишем наш пример с While на цикл for

for(int days = 0; days < 365; days++)
{
	Console.WriteLine("С начала года прошло" + days "дней");
	// Увеличивать внутри уже ничего не надо, мы это делаем в одном из блоков в голове цикла
}

// Можно задаться вопросом, а какая тогда хуй разница между ними, ЗОЧИМ ИХ 2 придумали. Ну тут дело вот в чем. Цикл for, может работать только с числовым счетчиком. Никаких других условий ему не надо. А вот цикл while может принимать на вход любое условное выражение и пока оно верно, он будет выоплняться.

bool isGameStarted = true;
while (isGameStarted)
{
	Console.WriteLine("Игра запущена"); // Если оставить так, то наш цикл будет работать вечно, так как мы не меняем перменную isGameStarted, она у нас всегда true.
}

bool isGameStarted = true;
```
